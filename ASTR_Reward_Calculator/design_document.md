# Astar dApp Staking リワード計算アプリ - 設計書

## 1. プロジェクト概要

**プロジェクト名**: ASTR Reward Tax Calculator  
**目的**: Astar Network の dApp Staking で獲得したリワード（ASTR）から得られた利益を計算し、日本の所得税確定申告対応のためのデータ管理ツール  
**実行環境**: ブラウザのみ（機密情報はローカルに保存）

---

## 2. 機能要件

### 2.1 基本機能（必須）

#### 2.1.1 データ入力画面
- **日付**: カレンダーピッカーで入力
- **Claim されたASTR量**: 数値入力（小数点第8位まで対応、Astar の最小単位対応）
- **Claim時のドル円為替レート**: 数値入力（小数点第2位まで）
- **Claim時のASTRレート（USD）**: 数値入力（小数点第4位まで）
- **トランザクションFee（ASTR）**: 数値入力（小数点第8位まで）

#### 2.1.2 データ表示機能
各Entry で以下を自動計算して表示：
- **クレーム前金額（USD）**: `ASTR量 × USDレート`
- **クレーム前金額（JPY）**: `ASTR量 × USDレート × ドル円レート`
- **利益（クレーム前後の差分考慮）**: `(ASTR量 - Fee) × USDレート - クレーム時のコスト`
- **利益（JPY）**: `利益（USD） × ドル円レート`

#### 2.1.3 集計表示
- **全クレーム数**
- **合計ASTR量**
- **合計Fee**
- **総リワード額（USD）**
- **総リワード額（JPY）**
- **集計期間**: 日付範囲を自動計算

#### 2.1.4 データ管理機能
- **追加**: 新規Entryを追加
- **編集**: 既存Entryを編集
- **削除**: 既存Entryを削除
- **並び替え**: 日付順・ASTR量順でソート可能

### 2.2 拡張機能（推奨）

#### 2.2.1 データ永続化
- ブラウザの **LocalStorage** を使用してデータを自動保存
- 自動保存の確認表示

#### 2.2.2 データのエクスポート/インポート
- **CSV形式**: Excelで編集可能な形式でエクスポート
- **JSON形式**: バックアップ・リストアに最適
- **PDF形式**: 確定申告用の帳票出力（要検討）

#### 2.2.3 複数通貨レート管理
- USD/JPY 以外の通貨対応（将来的な拡張性）
- レート履歴の保存機能

#### 2.2.4 確定申告補助機能
- **暦年集計**: 1月1日～12月31日の集計
- **事業年度別集計**: カスタム期間の集計
- **税務上の注記**: 雑所得か事業所得かの判定補助

## 5. データ構造

### 5.1 Entry（1件のClaimデータ）

```json
{
  "id": "uuid-1234",
  "date": "2024-03-05",
  "astrAmount": 50.12345678,
  "exchangeRateJPY": 150.50,
  "astrPriceUSD": 0.1234,
  "transactionFee": 0.5,
  "createdAt": "2024-03-05T10:30:00Z",
  "updatedAt": "2024-03-05T10:30:00Z"
}
```

### 5.2 計算フィールド（データベース非保存）

```javascript
{
  // Claim時点の価値
  rewardUSD: astrAmount × astrPriceUSD,
  rewardJPY: rewardUSD × exchangeRateJPY,
  
  // 実際の利益（Fee差し引き）
  profitUSD: (astrAmount - transactionFee) × astrPriceUSD,
  profitJPY: profitUSD × exchangeRateJPY,
  
  // Fee相当額
  feeUSD: transactionFee × astrPriceUSD,
  feeJPY: feeUSD × exchangeRateJPY
}
```

## 6. 計算ロジック

### 6.1 各Entry の計算式

| 項目 | 計算式 | 備考 |
|------|--------|------|
| リワード額（USD） | `ASTR量 × USDレート` | Claim時の価値 |
| リワード額（JPY） | `リワード額（USD） × ドル円レート` | - |
| Fee相当額（USD） | `Fee × USDレート` | 費用化可能部分 |
| Fee相当額（JPY） | `Fee相当額（USD） × ドル円レート` | - |
| **利益（USD）** | `(ASTR量 - Fee) × USDレート` | 雑所得換算 |
| **利益（JPY）** | `利益（USD） × ドル円レート` | 日本での所得 |

### 6.2 集計計算

```
合計リワード（USD） = Σ(各Entry のリワード額（USD）)
合計リワード（JPY） = Σ(各Entry のリワード額（JPY）)
合計Fee（USD） = Σ(各Entry のFee相当額（USD）)
合計Fee（JPY） = Σ(各Entry のFee相当額（JPY）)
```

## 7. 追加提案仕様

### 7.1 税務関連機能

#### 提案1: 税区分の判定
```
入力額の合計が年20万円以上 → 「事業所得の可能性あり」と表示
入力額が年20万円以下 → 「雑所得」と表示
※ ただし、詳細は税理士に相談するよう案内
```

#### 提案2: 所得税区分別集計
- **給与所得と分離**: リワード収入を分離計算
- **他の雑所得との合算**: オプションで他の仮想通貨収入も入力可能

#### 提案3: 経費計算補助
- **Gas代（トランザクションFee）**: 必定支出として費用計上
- **その他経費**: マイニング・ステーキング関連の費用を追加入力

### 7.2 取引分析機能

#### 提案4: 統計情報の表示
```
- 平均リワード（1回あたり）
- 平均USDレート
- 平均ドル円レート
- 最大リワード
- 最小リワード
- Claim頻度（日数平均）
```

#### 提案5: 時系列グラフ
```
- リワード額の推移（折れ線グラフ）
- ASTR/USD レートの推移
- ドル円レートの推移
- 累積利益（棒グラフ）
```

### 7.3 ユーザ支援機能

#### 提案6: テンプレート機能
```
- よく使う通貨レートを保存
- 同じレートで複数入力する場合は「テンプレート適用」
```

#### 提案7: 入力サジェスチョン
```
- 前回のドル円レートを入力値として提案
- 過去のASTRレートの平均値を提案
```

#### 提案8: フォーム検証とエラーチェック
```
- 負の値の入力禁止
- 日付の将来日を入力した場合の警告
- 異常に高い/低いレート入力時の確認
```

### 7.4 データ管理の堅牢性

#### 提案9: データバージョン管理
```
- LocalStorage に保存時、バージョン番号を付与
- 今後のスキーマ変更に対応
```

#### 提案10: データ検証とバックアップ
```
- 定期的なデータ整合性チェック
- 手動バックアップ機能
- インポート時の重複排除
```
